Recipes & Food Manager is a desktop application built in Python. It lets you build a personal database of food items and recipes, plan diets, and surface statistics about your spending and nutritional intake.

## Why it exists

The app was created to answer practical questions about personal health and spending:

- Where does grocery money actually go?
- How many calories and which nutrients am I consuming day-to-day?
- Is my diet balanced, and how can I adjust it?

Rather than relying on third-party services, it keeps all data local and gives you full control over your food and recipe information.

## Core entities

The application revolves around three main data types:

<AccordionGroup>
  <Accordion title="Food">
    A Food object represents a single ingredient or product. It stores:

    - Name and category
    - Price per unit
    - Calorie count
    - Macronutrient breakdown (protein, fat, carbohydrates)
  </Accordion>
  <Accordion title="Recipe">
    A Recipe is a collection of Food items with quantities and preparation instructions. It stores:

    - List of required ingredients with amounts
    - Estimated preparation time
    - Derived totals for cost, calories, and nutrients
  </Accordion>
  <Accordion title="Diet">
    A Diet is a plan composed of Food items and Recipes arranged over a time period. It enables:

    - Day-by-day meal planning
    - Summary statistics across the full plan
    - Comparison of nutritional targets against actual intake
  </Accordion>
</AccordionGroup>

| Module | Responsibility |
|---|---|

| `constants.py` | Derived constants computed from user settings |
| `settings.py` | Loads and merges `settings.yaml` with built-in defaults |
| `utils.py` | Shared utility helpers |

## Data storage

All application data — food items, recipes, and diet plans — is persisted to local JSON files. There is no external database or network dependency. This makes the app fully portable and easy to back up.

Configuration is managed through `src/settings.yaml`. On startup, `settings.py` merges your file with built-in defaults so that missing keys never cause a crash.

```yaml src/settings.yaml
template:
  property: 0    # int
property: 1
```

<Note>
  The `constants.py` module reads from the merged settings at import time, so any change to `settings.yaml` takes effect on the next application start.
</Note>
