Get the app running on your machine in a few minutes.

## Prerequisites

<Info>
  You need Python 3.10 or later. Check your version with `python --version` or `python3 --version`.
</Info>

The app depends on a small set of Python packages:

| Package | Purpose |
|---|---|
| `numpy` | Numerical calculations for nutrition and cost stats |
| `icecream` | Debug-friendly value printing during development |
| `customtkinter` | Modern GUI widgets (planned UI layer) |
| `pyyaml` | Reads the `settings.yaml` configuration file |

## Installation

<Steps>
  <Step title="Clone the repository">
    Download the source code to your machine.

    ```bash
    git clone https://github.com/Isma-GoPo/Recipes_and_Food-Manager.git
    cd Recipes_and_Food-Manager
    ```
  </Step>

  <Step title="Create a virtual environment (recommended)">
    Isolate the project dependencies from your system Python.

    <CodeGroup>
    ```bash macOS / Linux
    python3 -m venv .venv
    source .venv/bin/activate
    ```
    ```bash Windows
    python -m venv .venv
    .venv\Scripts\activate
    ```
    </CodeGroup>

    <Tip>
      Activate the virtual environment every time you return to the project.
    </Tip>
  </Step>

  <Step title="Install dependencies">
    ```bash
    pip install numpy icecream customtkinter pyyaml
    ```
  </Step>

  <Step title="Run the application">
    Launch the entry point from the `src/` directory so that relative imports resolve correctly.

    ```bash
    cd src
    python main.py
    ```
  </Step>
</Steps>

## What happens on startup

When you run `main.py`, the application:

1. Imports `settings.py`, which reads `src/settings.yaml` and merges it with the built-in default values.
2. Imports `constants.py`, which derives typed constants from the merged settings.
3. Imports `functions.py` and the `utils` package.
4. Enters the main program loop (the interface is under active development).

If `settings.yaml` is missing or has incomplete keys, the app falls back to sensible defaults defined in `settings.py` — so it will not crash on first run.

## Configuration

The default `src/settings.yaml` looks like this:

```yaml src/settings.yaml
template:
  property: 0    # int
property: 1
```

You can edit this file to override default values. See the [configuration guide](/configuration/settings) for all available options.

## Troubleshooting

<AccordionGroup>
  <Accordion title="ModuleNotFoundError: No module named 'yaml'">
    The `pyyaml` package is not installed. Run `pip install pyyaml` and try again.
  </Accordion>
  <Accordion title="ModuleNotFoundError: No module named 'icecream'">
    Run `pip install icecream` to add the missing package.
  </Accordion>
  <Accordion title="FileNotFoundError when loading settings.yaml">
    Make sure you run `python main.py` from inside the `src/` directory. The settings loader uses a relative path (`src\settings.yaml`).
  </Accordion>
</AccordionGroup>
